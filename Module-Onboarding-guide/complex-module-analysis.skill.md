---
name: "Complex Module Analysis Skill"
description: "Reusable skill for analyzing nested module hierarchies, workflow branches, and transaction categories in large repositories."
---

This skill is intended for onboarding and impact analysis inside a large module tree.

## Capabilities
- Identify module boundaries and nested submodule structure
- Detect workflow branches and business paths
- Compare transaction flows such as frontbook vs backbook
- Highlight integration points and likely impact zones
- Translate technical structure into analyst-friendly business language

## When to use
- when the module has multiple branches or nested workflows
- when you need a clear tree map of a complex code area
- when you want a quick summary of where to start changes

## Execution
1. Ask for the exact module path if not provided.
2. Build a concise tree of submodules and branches.
3. Mark the main business workflows and their triggers.
4. Identify where branch-specific logic lives.
5. Summarize in plain-language bullets with clear starting points.
